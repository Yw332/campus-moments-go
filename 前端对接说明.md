# 前端对接说明文档

## 🌐 API服务器信息

**服务器地址**: `http://106.52.165.122:8080`

**健康检查**: `GET http://106.52.165.122:8080/health`

## 📋 主要接口清单（概览）

### 🔐 用户认证
- `POST /auth/register` - 用户注册
- `POST /auth/login` - 用户登录（返回 JWT）
- `POST /api/auth/logout` - 退出登录
- `GET /api/users/profile` - 获取用户资料（需认证）
- `PUT /api/users/password` - 修改密码（需认证）

### 📄 动态（Moments）
- `GET /moments` - 获取动态列表（公开，无需认证）
- `POST /api/moments` - 发布动态（需认证）
- `GET /api/moments/:id` - 获取动态详情（公开/私有鉴权）
- `PATCH /api/moments/:id` - 编辑动态（需认证，需为作者）
- `DELETE /api/moments/:id` - 删除动态（需认证，需为作者或管理员）

### 🧰 管理后台（Admin）
- `POST /admin/login` - 管理员登录（返回 JWT）
- `GET /admin/menu` - 管理员侧边菜单（需管理员权限）
- `GET /admin/users?page=&pageSize=` - 列出用户（需管理员权限）
- `GET /admin/users/:id` - 获取用户详情（需管理员权限）
- `POST /admin/users/:id/password` - 管理员重置用户密码（需确认参数，需管理员权限）
- `DELETE /admin/users/:id?confirm=true` - 删除用户（需管理员权限，需 confirm=true）
- `GET /admin/contents?page=&pageSize=` - 列出内容/帖子（需管理员权限）
- `GET /admin/contents/:id` - 内容详情（需管理员权限）
- `DELETE /admin/contents/:id?confirm=true` - 删除内容（需管理员权限，需 confirm=true）

### 📤 文件上传
- `POST /api/upload/file` - 通用文件上传（需认证）
- `POST /api/upload/avatar` - 头像上传（需认证）

## 🔑 鉴权 & 请求头（重要）

所有需要认证的接口（通常以 `/api/` 或 `/admin/` 开头）要求在请求头中携带JWT：

```http
Authorization: Bearer <token>
Content-Type: application/json
```

- 登录接口 `/auth/login` 与 `/admin/login` 返回 JSON 中 `data.token`。
- Token 在 payload 中包含 `userId`（用户登录）或 `username`（管理员登录）等字段，服务端会基于 `username` 在 `admins` 表判断管理员权限。
- 管理员 token 与用户 token 使用同一 JWT 格式（仅 claims 不同）。

## 📦 请求/响应示例（含管理员场景）

### 1) 用户登录
请求：
```http
POST /auth/login
Content-Type: application/json

{ "account": "itestuser", "password": "TestPass123" }
```
响应：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "<jwt_token>",
    "userInfo": { "userId": "0000000011", "username": "itestuser", "phone": "138xxxx" }
  }
}
```

### 2) 管理员登录
请求：
```http
POST /admin/login
Content-Type: application/json

{ "account": "itestadmin", "password": "Admin12345" }
```
响应：
```json
{ "code": 200, "message": "登录成功", "data": { "token": "<admin_jwt_token>" } }
```
说明：管理员 token 在后续请求中用于访问 `/admin/*` 接口。

### 3) 管理：列出用户
请求：
```http
GET /admin/users?page=1&pageSize=10
Authorization: Bearer <admin_token>
```
响应（示例）：
```json
{
  "code": 200,
  "message": "ok",
  "data": [
    { "userId": "0000000001", "username": "张三", "phone": "13800138001", "status": 1 },
    ...
  ]
}
```

### 4) 管理：重置用户密码
请求：
```http
POST /admin/users/11/password
Authorization: Bearer <admin_token>
Content-Type: application/json

{ "newPassword": "NewPass123", "confirm": true }
```
响应：
```json
{ "code": 200, "message": "重置成功", "data": null }
```

### 5) 管理：删除用户（注意外键相关）
请求（必须带 confirm=true）：
```http
DELETE /admin/users/11?confirm=true
Authorization: Bearer <admin_token>
```
行为说明：删除用户之前后端会先删除该用户的 `moments`（动态），以避免外键约束阻止删除，整个过程在事务中执行。

### 6) 创建动态（用户）
请求：
```http
POST /api/moments
Authorization: Bearer <user_token>
Content-Type: application/json

{
  "content": "今天学习了 Go 和 Gin",
  "tags": ["学习","后端"],
  "media": null
}
```
响应：
```json
{
  "code": 200,
  "message": "发布成功",
  "data": {
    "id": 123,
    "authorId": "0000000011",
    "content": "今天学习了 Go 和 Gin",
    "tags": ["学习","后端"],
    "status": 1
  }
}
```

## 📌 对接细节（注意点）
- ID 格式：数据库中的 `users.id` 为 **10 位数字字符串**（例如 `0000000011`），但大部分 API 支持前端以数字形式传递（例如 `/admin/users/11`），后端会自动补零处理；建议前端以数字或字符串都可，API 会兼容。
- 分页参数：`page`（>=1）与 `pageSize`（默认10），返回格式通常在 `data` 中直接为数组或包含 `pagination` 字段。
- 删除操作：对会产生数据级联或外键依赖的删除，后端通常要求 `?confirm=true` 防止误删。
- 鉴权失败：当请求缺少或 token 无效会返回 `401`；管理员权限不足会返回 `403`。
- 文件/媒体字段：`media` 字段使用 JSON（数组或 null），图片应先上传到 `/api/upload/file` 获取 URL，再放入 `media`。
- CORS：开发环境已允许常见前端 origin；若需要特定域名白名单请提出，我可以添加中间件限制。

## 🐞 常见错误 & 处理
- "missing authorization header" → 请带 Authorization: Bearer <token>
- 400 参数错误 → 请检查 body、必填项与字段类型
- 403 管理员权限不足 → 使用管理员账号（或确认 token 对应 username 在 `admins` 表存在）
- 500 服务器错误 → 请将请求与时间点日志提供给后端以便排查

## 📱 快速示例代码（常用）
### 通用认证请求封装（JS）
```javascript
const authRequest = async (path, { method = 'GET', body } = {}) => {
  const token = localStorage.getItem('token');
  const headers = { 'Content-Type': 'application/json' };
  if (token) headers['Authorization'] = 'Bearer ' + token;

  const res = await fetch('http://106.52.165.122:8080' + path, {
    method,
    headers,
    body: body ? JSON.stringify(body) : undefined
  });
  return await res.json();
};

// 管理员登录示例
const adminLogin = async () => {
  const res = await fetch('http://106.52.165.122:8080/admin/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ account: 'itestadmin', password: 'Admin12345' })
  });
  return await res.json();
};
```
