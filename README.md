# campus-moments
软件工程大作业仓库
课程项目内容：
本项目拟定开发一个具有展示功能的、类似朋友圈的多媒体展示小软件。该软件包含一个用户使用的小程序前端和一个管理员使用的Web后台。

多媒体朋友圈展示系统
1. 产品概述
1.1 产品背景与目标
本项目旨在开发一个具有展示功能的、类似朋友圈的多媒体展示平台。该平台由一个小程序前端和一个Web管理后台构成，旨在为用户提供一个安全、便捷、有趣的分享空间，并为管理员提供高效的内容与用户管理工具。

1.2 用户画像
终端用户：需要记录和分享生活、学习或工作内容的普通用户（如学生、白领等）。
管理员：系统运营人员，负责用户管理、内容审核、数据监控。
1.3 产品功能矩阵
模块 核心功能（MVP） 增强功能（加分项）

小程序前端 登录/注册、多媒体（图文/视频）发布、个人内容CRUD、标签管理、浏览与评论 好友关系、敏感词过滤、视频流处理、AI内容处理

Web管理后台 管理员登录、用户管理、内容管理 数据统计分析图表、高级审核机制

2. 系统架构与模块说明
系统主要分为两个部分：

2.1 小程序前端
1.  登录注册模块
2.  主展示/首页模块
3.  个人中心模块

2.2 Web管理后台
1.  管理员登录模块
2.  用户管理模块
3.  内容管理模块
4.  数据统计模块（线框图中未直接体现，根据需求补充）


3. 功能需求详述
3.1 小程序前端
3.1.1 登录注册模块
描述：用户入口，提供注册和登录功能。
功能流程：
    1.  登录：用户输入用户名/手机号和密码进行登录。
    2.  注册：用户设置用户名、手机号、密码等信息完成账号注册。
3.1.2 主页
描述：内容展示的核心页面，以信息流形式呈现用户及其好友发布的内容。
核心功能：
浏览信息流：查看按时间倒序排列的多媒体内容（图片、视频、文本）。
内容互动：对内容进行点赞、评分、评论。
内容搜索/筛选：支持通过关键词、标签、日期进行内容检索。
3.1.3 我的（个人中心）
描述：用户管理个人资料和内容的中心。
核心功能：
内容管理：查看“我发布的”所有内容，并支持对自有内容进行修改、删除。
资料设置：修改个人信息、头像、登录密码等。
3.2 Web管理后台

3.2.1 管理员登录
描述：管理员入口，确保后台安全。
功能：输入管理员账号和密码登录系统。
3.2.2 用户管理
描述：系统管理员对所有注册用户进行管理的面板。
核心功能：
用户查询：支持按用户ID、用户名、注册时间等进行搜索和筛选。
用户信息查看：查看用户基本信息及其活动概况。
用户操作：对违规用户进行删除、禁言等管理操作。
3.2.3 内容管理
描述：系统管理员对所有用户发布的内容进行管理的面板。
核心功能：
内容查询：支持按内容ID、类型、发布者、状态（如待审核、已审核、违规）等进行筛选。
内容审核：查看内容详情（图片、视频、文本）。
内容操作：对违规或不良内容进行删除、屏蔽等操作。
3.2.4 数据统计（需补充的页面）
描述：以图表形式可视化系统运行数据，辅助运营决策。
核心功能：
    用户增长曲线图（日/周/月）。
内容发布量统计图。
用户活跃度（DAU/MAU）仪表盘。
热门标签排行榜。

4. 非功能性需求
1.  性能：小程序页面加载时间应小于2秒，图片视频加载流畅。
2.  安全性：用户密码需加密存储，管理后台需有权限验证，防止越权操作。
3.  可用性：界面简洁直观，符合用户使用习惯，操作路径清晰。
4.  兼容性：小程序需兼容主流微信版本。Web后台需兼容Chrome、Safari等主流浏览器。
5. 后续迭代与加分项规划
1.  社交功能：在“我的”页面增加“好友”或“关注”模块，实现好友添加、私信等功能。
2.  内容安全：集成敏感词过滤系统，在内容发布和评论时自动过滤。
3.  AI集成：
    视频处理：自动生成封面、关键帧提取。
